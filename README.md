# Financial Transaction Anomaly & Risk Monitoring System

## Project Overview
This is a **Python + SQL** project that simulates a real banking system for detecting suspicious financial activities. 

It can:
- Generate fake bank transaction data
- Store the data in a database
- Automatically detect high-risk anomalies like **structuring** (smurfing) and **velocity spikes**
- Flag suspicious transactions and create a report

This type of system is used in real banks for **Anti-Money Laundering (AML)** compliance.

## Features
- Generates realistic synthetic transaction data
- Uses SQLite database to store accounts and transactions
- Detects anomalies using SQL queries and Python analysis
- Saves flagged high-risk transactions to a CSV file
- Easy to understand and extend

## Technologies Used
- **Python** — Main programming language
- **Pandas & NumPy** — For data analysis
- **SQLite** — Lightweight database
- **SQL** — For efficient data querying
